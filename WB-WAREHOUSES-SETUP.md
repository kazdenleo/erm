# 📦 Настройка системы складов Wildberries

## Обзор

Система управления складами WB позволяет:
- **Загружать данные о складах WB** раз в сутки через API
- **Сопоставлять внутренние склады** с WB складами
- **Использовать реальные тарифы** для точного расчета цен
- **Автоматически обновлять кэш** данных о складах

## 🚀 Быстрый старт

### 1. Настройка API ключа WB

1. Откройте приложение: **http://localhost:3001**
2. Перейдите в **"Интеграции"** → **"Маркетплейсы"** → **"Wildberries"**
3. Введите ваш **API ключ WB**
4. Нажмите **"Сохранить"**

### 2. Загрузка данных о складах

1. Перейдите в **"Склады"** → **"📦 WB Склады"**
2. Нажмите **"🔄 Обновить данные WB"**
3. Дождитесь загрузки списка складов WB

### 3. Сопоставление складов

1. В разделе **"Сопоставление складов"** выберите:
   - **Внутренний склад** (из вашей ERP)
   - **Склад WB** (из загруженного списка)
2. Нажмите **"Сохранить"**

## 🔧 Детальная настройка

### API Endpoints

#### Получение складов WB
```http
GET /api/wb-warehouses
```

#### Обновление кэша складов
```http
POST /api/wb-warehouses/refresh
```

#### Статус кэша
```http
GET /api/wb-warehouses/status
```

#### Сопоставления складов
```http
GET /api/warehouse-mappings
POST /api/warehouse-mappings
```

## ⚙️ Автоматическое обновление

### Настройка Task Scheduler (Windows)

1. Откройте **"Планировщик заданий Windows"**
2. Создайте **"Простое задание"**
3. Настройте:
   - **Имя**: "WB Warehouses Cache Update"
   - **Триггер**: Ежедневно в 02:00
   - **Действие**: Запуск программы
   - **Программа**: `C:\Users\Денис\testCursor\update-wb-warehouses-cache.bat`

### Ручное обновление

```bash
# Запуск скрипта обновления
node update-wb-warehouses-cache.js

# Или через batch файл
update-wb-warehouses-cache.bat
```

## 📊 Использование в расчете цен

### Передача ID склада в API

При расчете цен WB можно передать ID склада:

```javascript
// Пример запроса с указанием склада
const response = await fetch(`/product/prices/wb?offer_id=AN1048&category_id=5409&warehouse_id=stock_1234567890`);
```

### Логика выбора склада

1. **Если передан `warehouse_id`**:
   - Ищется сопоставление в `warehouseMappings.json`
   - Используется соответствующий склад WB
   - Применяются тарифы этого склада

2. **Если сопоставление не найдено**:
   - Используется первый доступный склад WB
   - Применяются базовые тарифы

3. **Если нет данных о складах**:
   - Возвращается ошибка "Данные недоступны"

## 🔍 Мониторинг и отладка

### Проверка статуса кэша

```javascript
// Проверка статуса кэша складов
const response = await fetch('/api/wb-warehouses/status');
const data = await response.json();

console.log('Кэш складов:', {
  exists: data.data.exists,
  size: data.data.size,
  lastModified: data.data.lastModified
});
```

## 🚨 Устранение неполадок

### Проблема: "Нет данных о складах WB"

**Причины:**
- API ключ WB не настроен
- Ошибка подключения к API WB
- Кэш не загружен

**Решение:**
1. Проверьте настройки API ключа WB
2. Нажмите "🔄 Обновить данные WB"
3. Проверьте логи сервера

### Проблема: "Склад не найден"

**Причины:**
- Сопоставление не настроено
- Неверный ID склада

**Решение:**
1. Перейдите в "Склады" → "📦 WB Склады"
2. Настройте сопоставление складов
3. Проверьте ID склада в запросе

### Проблема: "Тарифы не применяются"

**Причины:**
- Данные о тарифах отсутствуют
- Неверная структура данных

**Решение:**
1. Обновите кэш складов WB
2. Проверьте структуру данных в логах
3. Убедитесь, что API возвращает корректные тарифы
