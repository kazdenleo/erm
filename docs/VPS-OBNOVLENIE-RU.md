# Как обновить программу на VPS — простой алгоритм

Читайте **сверху вниз**. Команды копируйте в **терминал на сервере** (SSH), не в WinSCP.

Сначала **один раз** впишите свои значения (чтобы не искать каждый раз):

| Что | Ваше значение (пример) |
|-----|-------------------------|
| Папка проекта на сервере | `/opt/erm` |
| Имя процесса в PM2 | `erm-api` (смотрите командой `pm2 list`) |
| Имя базы PostgreSQL | `erp_system` |

Дальше в командах мы пишем `/opt/erm` и `erm-api` — **замените на свои**, если у вас другие.

---

## Словарь в пяти фразах

- **Git / push / pull** — вы сохраняете код у себя и **отправляете на GitHub** (push); на сервере **скачиваете этот же код** (pull). Так на VPS оказывается та же версия, что и у вас на ПК.
- **VPS** — ваш удалённый сервер (Beget), где крутится сайт и база.
- **API** — программа на сервере (Node.js), которая отвечает сайту. Её перезапускают после обновления кода — чаще всего через **PM2**.
- **Миграция** — маленький SQL-скрипт: добавить таблицу или колонку. Запускается **на сервере** после `git pull`, командой `npm run migrate`.
- **`.env`** — файл с паролями и настройками **на сервере**; в Git обычно не попадает. Новую строку из примера в проекте — дописываете в серверный `.env` руками.

## Алгоритм 1. Обновили только код (без изменений в базе)

Делайте по шагам **по порядку**.

### Шаг 1 — на своём компьютере

1. Сохранили работу в Git и отправили на GitHub:

   ```text
   git add .
   git commit -m "коротко что сделали"
   git push
   ```

   **Зачем:** без push на сервере после `git pull` не появятся ваши последние файлы.

### Шаг 2 — зайти на сервер по SSH

Откройте консоль / PuTTY / «SSH» в панели хостинга и войдите на VPS.

**Зачем:** обновление командами делается только так; в окне с папками (как на скрине) этих команд нет.

### Шаг 3 — скачать свежий код

```bash
cd /opt/erm
git pull
```

**Зачем:** подтянуть с GitHub то, что вы запушили на шаге 1.

### Шаг 4 — только если менялись зависимости

Если в коммите трогали `server/package.json` (новые библиотеки), выполните:

```bash
cd /opt/erm/server
npm ci
```

**Зачем:** установить те же версии пакетов, что указаны в проекте. Если `package.json` не меняли — шаг можно **пропустить**.

### Шаг 5 — только если собираете сайт (React) на сервере

Если у вас принято собирать клиент на VPS:

```bash
cd /opt/erm/client
npm ci
npm run build
```

**Зачем:** браузер получает уже **собранные** файлы. Если фронт не собираете на сервере — пропустите.

### Шаг 6 — перезапустить программу

```bash
pm2 restart erm-api
pm2 save
```

**Зачем:** Node подхватит новый код. `pm2 save` — чтобы после перезагрузки сервера процесс снова поднялся.

### Шаг 7 — проверка

Откройте сайт в браузере, зайдите в программу, проверьте то, что меняли.

Если «белый экран» или 502 — смотрите логи:

```bash
pm2 logs erm-api --lines 50
```

## Алгоритм 2. Менялась база (добавили файл в `server/scripts/migrations/sql/`)

Сделайте **всё из алгоритма 1**, но **между шагом 3 и шагом 4** (после `git pull`) вставьте блок про базу.

### Доп. шаг А — резервная копия базы (страховка)

```bash
sudo -u postgres pg_dump -Fc erp_system > /root/backup_$(date +%F_%H%M).dump
```

**Зачем:** если миграция ошибётся, есть файл, из которого можно восстановить БД. Имя базы `erp_system` замените на своё, если другое.

### Доп. шаг Б — применить новые миграции

```bash
cd /opt/erm/server
npm run migrate
```

**Зачем:** выполнятся только **ещё не применённые** скрипты (старые повторно не трогают).

Проверить список:

```bash
npm run migrate:status
```

### Потом

Снова **шаги 4–7** из алгоритма 1 (при необходимости `npm ci`, сборка клиента, **`pm2 restart`**).

## Одна страница: что нажимать, если забыли

| Ситуация | Действие |
|----------|----------|
| Только правки в коде | ПК: push → VPS: `cd ... && git pull` → `pm2 restart ...` |
| Плюс новые `.sql` миграции | После `git pull`: `pg_dump` → `npm run migrate` → `pm2 restart` |
| Меняли `package.json` | После `git pull`: `npm ci` в `server/` (и в `client/`, если нужно) |
| Не знаете имя в PM2 | На VPS: `pm2 list`, колонка `name` |
| Не знаете папку проекта | На VPS: `ls /opt` или спросить, куда ставили клон Git |

## Как узнать имя процесса PM2 и папку проекта (один раз)

На сервере по SSH:

```bash
pm2 list
```

Запомните имя в колонке **name** (подставляете вместо `erm-api`).

Папка проекта — та, где есть `server/` и обычно `.git`:

```bash
cd /путь/который/знаете
ls
# должны увидеть server, возможно client
```

---

## Если совсем непонятно, что такое API

Не нужно ничего отдельно «включать». **API** здесь = тот самый процесс **Node.js**, который вы перезапускаете командой **`pm2 restart`**. Сайт в браузере к нему обращается сам. Вам достаточно после обновления кода **перезапустить этот процесс** (и при изменении БД — прогнать миграции).
